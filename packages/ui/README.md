# @ps/ui

跨应用复用的基础展示层包。

## 职责

- 提供 `web` 和 `admin` 都会复用的基础 UI 组件或样式。
- 保持组件中性、轻业务、可组合。

## 当前内容

- `Logo`
- `styles.css`

这说明当前 `ui` 仍然是一个很薄的包，后续只在出现真实双端复用时再继续扩展。

## 适合放入的内容

- 基础展示组件
- 低业务耦合的通用 UI 原子组件
- 共享设计 token 或基础样式

## 不适合放入的内容

- `SiteHeader` / `SiteFooter` 这类站点结构组件
- `ArticleForm` 这类后台业务表单
- 依赖服务端数据访问的组件
- 某个 app 专属的交互编排

## 收敛规则

- 先在 app 内落地，确认 `web` 和 `admin` 都要用，再考虑上收
- 没有真实消费者的组件不要预先放进来
- 如果组件需要写死某个 app 的资源路径、文案或页面结构，通常不应该放在这里

## 导入约束

- 组件 API 尽量保持中性
- 应用资源路径、业务文案、页面级布局由使用方提供
- 不为了“看起来像组件库”而提前堆组件
